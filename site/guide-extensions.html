제품 유형 자가진단

아래 4가지 질문에 "예"라고 답하는 항목에 해당하는 확장 섹션을 읽으세요. 해당 없는 섹션은 건너뛰어도 됩니다.

일반 소비자 대상 서비스(B2C)인가?
예 → 브랜드 시각 언어, 인터랙션/모션 품질 섹션 적용
다국어 지원이 필요한가?
예 → 다국어(i18n) 설계 섹션 적용
다크모드를 지원할 계획인가?
예 → 다크모드 대응 섹션 적용
PC + 모바일 웹 기반 서비스인가?
예 → 반응형 + 기기 QA 섹션 적용
📌

4가지 모두 해당한다면 이 가이드의 확장 섹션 전체가 필요한 B2C 글로벌 웹 서비스입니다. 신규 서비스 구축 시 Core + 확장 전체를 AI 컨텍스트 문서에 포함하세요.

시맨틱 토큰 레이어

색상값을 직접 쓰는 대신 목적 기반 토큰으로 추상화합니다. 테마가 바뀌어도 컴포넌트 코드는 그대로 유지됩니다.

하지 말 것
background: #FFFFFF;
color: #090C32;
border: 1px solid #E0E2ED;
이렇게
background: var(--bg-card);
color: var(--text-primary);
border: 1px solid var(--border);

CSS 토큰 구조

tokens.css
:root {
  /* 배경 */
  --bg: #F7F8FC;
  --bg-card: #FFFFFF;
  --bg-code: #ECEDF5;

  /* 테두리 */
  --border: #E0E2ED;
  --border-light: #ECEEF5;

  /* 텍스트 */
  --text-primary: #090C32;
  --text-secondary: #464A65;
  --text-muted: #9398AC;

  /* 브랜드 */
  --accent: #664FFF;
  --accent-dim: rgba(102, 79, 255, 0.08);
}

[data-theme="dark"] {
  --bg: #0C0E1A;
  --bg-card: #141728;
  --bg-code: #1B1E30;

  --border: #262A42;
  --border-light: #1D2038;

  --text-primary: #DFE2F0;
  --text-secondary: #8B93B4;
  --text-muted: #4D5470;

  --accent: #7A6FFF;
  --accent-dim: rgba(122, 111, 255, 0.15);
}

전환 방식

  • 기본값 — prefers-color-scheme: dark 시스템 설정 따름
  • 수동 전환 — 사용자가 토글 시 localStorage에 저장, 이후 방문 시 유지
  • FOUC 방지 — <head> 안 인라인 스크립트로 렌더 전 테마 즉시 적용
HTML <head> — FOUC 방지
<script>
(function(){
  var t = localStorage.getItem('theme')
       || (window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light');
  document.documentElement.setAttribute('data-theme', t);
})();
</script>

다국어(i18n) 설계

다국어 지원 예정 글로벌 서비스 해당하지 않으면 건너뛰어도 됩니다.

핵심 원칙

  • 텍스트 하드코딩 금지 — 모든 노출 문자열은 locale 파일에서 관리. 컴포넌트에 한국어 직접 입력 금지
  • 레이아웃은 텍스트 길이에 유연하게 — 영어는 한국어보다 평균 30~40% 길 수 있음. 고정 너비 버튼·레이블 주의
  • 폰트는 언어별로 — 한국어 Pretendard, 영어 Inter/system-ui 등 언어별 폰트 스택 분리

파일 구조

  ko.json       # 한국어 (기준 언어)
  en.json       # 영어
  ja.json       # 일본어 (추후)

# 다국어 작업 기준

## 절대 규칙
- 컴포넌트에 한국어/영어 직접 입력 금지
- 모든 텍스트는 locales/{lang}.json 키 참조
- 새 문자열 추가 시 ko.json 먼저 작성 후 en.json 동기화

## 레이아웃 주의사항
- 버튼 너비: min-width 기준, 텍스트 길이 맞춰 유연하게
- 영어 텍스트: 한국어 대비 최대 1.4배 너비 여유 확보
- 줄바꿈: 언어별 단어 경계가 다름, overflow 처리 필수

기기별 엣지케이스 체크리스트

  • iOS Safe Area — env(safe-area-inset-*) 적용. Dynamic Island(iPhone 14 Pro+), 홈 인디케이터 영역 침범 여부
  • Android Punch-hole — 상단 카메라 영역 겹침. viewport-fit=cover + safe-area 처리 동일 적용
  • 모바일 주소창 높이 — 100vh 대신 100dvh 사용. 주소창 show/hide 시 레이아웃 튀는 현상 방지
  • 터치 타겟 — 최소 44×44px. 버튼, 링크, 토글 전부 해당. 손가락 오탭 방지
  • 가로 모드 — landscape 전환 시 레이아웃 깨짐 여부 확인
  • 텍스트 확대 — OS 텍스트 크기 150% 설정 시 레이아웃 유지 여부

인터랙션 / 모션 품질

모션 원칙 3가지

01
목적이 있는 움직임
장식이 아닌 상태 변화 전달. 어디서 왔고 어디로 가는지 방향을 알 수 있어야 함
02
자연스러운 가속/감속
linear 지양. ease-out(진입), ease-in-out(전환)으로 물리적 자연감 부여
03
간결한 지속시간
짧고 빠르게. 불필요하게 긴 애니메이션은 사용자를 기다리게 함

브랜드 시각 언어

AI 결과물이 브랜드에서 이탈하는 패턴

  • 토큰 무시 — 시맨틱 토큰 대신 임의 hex 사용. 테마 일관성 붕괴
  • 폰트 혼용 — 정의된 폰트 스택 외 임의 폰트 추가
  • 간격 임의 설정 — 스페이싱 시스템 무시, px 값 자유 지정
  • 컴포넌트 재발명 — 이미 있는 컴포넌트를 새로 만들거나 변형
  • 모션 과잉 — 브랜드 톤과 맞지 않는 과도한 애니메이션

브랜드 토큰 정의

디자인 시스템의 핵심 가치를 토큰으로 고정합니다. AI가 이 토큰 밖에서 임의 판단하지 못하도록 규칙으로 명시합니다.

docs/brand-tokens.md
# 브랜드 토큰 정의

## 컬러
- Primary: var(--accent) — #664FFF
- 임의 컬러 추가 금지. 새 컬러 필요 시 디자이너 승인 후 토큰 추가

## 타이포그래피
- 본문: Pretendard 16px / line-height 1.75
- 헤딩: font-weight 700, letter-spacing -0.02em
- 코드: JetBrains Mono 13px

## 스페이싱
- 기준 단위: 4px (4, 8, 12, 16, 20, 24, 32, 40, 48, 64)
- 임의 px 값 금지

## 반경
- 카드: 10px (--radius)
- 인라인: 6px (--radius-sm)
- 알약형: 20px

AI 결과물 브랜드 체크리스트

  • 색상이 모두 토큰 변수로 지정되어 있는가
  • 폰트가 정의된 스택만 사용하는가
  • 간격이 스페이싱 단위 기준인가 (4의 배수)
  • 기존 컴포넌트를 재사용했는가, 새로 만들었다면 이유가 있는가
  • 다크모드에서도 동일하게 읽히는가
  • 이 화면이 우리 제품처럼 보이는가 — 마지막 감각적 판단
AI Workflow 문서는 AI에게 일을 시키기 위한 프롬프트 모음이 아니라, AI가 제품과 디자인 시스템을 이해하고 일관되게 작업하도록 만드는 운영 체계다.

프로덕트 디자이너가 집중해야 할 부분은 코드를 직접 잘 짜는 것이 아니라, AI가 흔들리지 않고 디자인 판단을 보조할 수 있도록 제품 맥락, UX 의도, 컴포넌트 사용 기준, 검토 기준을 구조화하는 것입니다.
