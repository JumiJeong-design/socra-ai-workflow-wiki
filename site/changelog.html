  • 모바일 대응 — 비교표가 작은 화면에서 한 열로 떨어지도록 보정

사이트 그룹 페이지 분리 · 검색 · UX 개선 2026.06.04

단일 페이지였던 ai-workflow-guide.html을 사이드바 그룹 단위로 분리하고, 검색 하이라이트와 scrollspy를 추가해 탐색 경험을 개선했습니다.

구조 변경

  • 사이트 그룹 페이지 분리 — 3000줄짜리 단일 페이지를 사이드바 그룹 단위 5개(basics/setup/build/ops/extensions)로 분리. wiki 문서도 실무가이드/플레이북 2개 그룹 페이지로 통합
  • prev/next 네비게이션 — 각 페이지 하단에 이전/다음 페이지 이동 버튼 추가. 전체 12개 페이지가 순서대로 연결
  • 빌드 스크립트 추가 — scripts/build-guide.py, scripts/build-wiki.py로 MD 수정 시 재빌드

검색 개선

  • 검색어 하이라이트 — 검색 결과 클릭 후 페이지 내 일치 텍스트 전체 하이라이트
  • 결과 간 이동 네비게이터 — 하단 플로팅 바로 ↑↓ 이동, 현재 위치(N/전체) 표시, ✕ 닫기

사이드바 UX

  • Scrollspy — 스크롤 위치 기반으로 현재 섹션 sidebar 링크 active 표시
  • 현재 페이지 그룹 표시 — 현재 보고 있는 페이지 그룹의 nav-label이 accent 컬러로 강조

콘텐츠 · 링크 정리

  • wiki 문서 한국어 번역 — 영어로 남아있던 섹션 전체 한국어로 수정. "역사적 교훈" → "시행착오 정리" 등 자연스러운 표현으로 개선
  • riiid/lampro → riiid/prism — 통합된 패키지 레포 링크로 업데이트
  • 불필요한 내용 제거 — Pending placeholder, 팀 역할 분담 테이블, 중복 divider 제거
  • index.html 링크 전수 업데이트 — 분리된 페이지 구조에 맞게 전체 내부 링크 수정
  • 공통 sidebar 캐시 버전을 v0.15로 갱신
0.14

AI 워크플로우 시각화 · 역할 구분 명확화 2026.06.02

운영 흐름을 인터랙티브 시각화 페이지로 분리하고, 각 확정 기준 표면별로 누가 작성하고 누가 리뷰하는지 역할을 명확히 정리했습니다.

신규 페이지

  • 워크플로우 시각화 페이지 추가 — 탐색 → 확정 기준 → 화면 설계 → 제품 반영 → QA → 기록&보강 전체 파이프라인을 시각화. 홈 대표 운영 흐름 요약을 링크로 교체

운영 흐름 개선

  • 역할 구분 명확화 — Figma(디자이너), Git(디자이너 md spec + 개발자 코드), Storybook(디자이너 story 기준 + 개발자 구현), Worklog(현재 디자이너)로 작성·리뷰 주체 정리
  • QA 피드백 루프 시각화 — Storybook 검수 → 제품 반영 → App QA → Issue/Worklog → Figma·Git·Storybook 보강으로 돌아오는 루프 명시
  • 레거시 vs 신규 플로우 비교표 추가 — 각각 장단점 정리. 이터레이션이 사이클마다 빨라지는 구조가 신규 플로우의 핵심 강점으로 정의
  • 화면 설계 단계 미결 항목 추가 — 컴포넌트 이상의 화면 조합·UX 흐름 단계에 동일 루프 적용 가능 여부는 테스트 필요로 표시
  • LNB 캐시 갱신 — 공통 sidebar 로딩 버전을 v0.14로 올려 최신 구성이 바로 반영되도록 수정
0.13

탐색 도구와 Source of Truth 분리 2026.06.02

디자인 탐색은 Figma뿐 아니라 코드 프로토타입, Storybook screen, local app 같은 실행 가능한 표면에서도 시작할 수 있도록 운영 흐름을 확장했습니다.

운영 루프 개선

  • 탐색 도구 확장 — Figma-first와 code-prototype-first 탐색을 모두 허용하고, 문제 성격에 따라 빠른 도구를 선택하도록 정리
  • SOT 역할 분리 — 시각 기준은 Figma, 구현 기준은 Git, 구현 검수는 Storybook, 맥락 기록은 Notes/Worklog로 구분
  • 프로토타입 정의 확장 — Figma Prototype뿐 아니라 HTML/React prototype, Storybook screen, local app도 실행 가능한 검수 표면으로 포함
  • LNB 캐시 갱신 — 공통 sidebar 로딩 버전을 v0.13으로 올려 최신 홈 구성이 바로 반영되도록 수정

0.7

실제 운영 도구 기준으로 재정리 2026.05.29

디자인팀이 실제로 사용할 도구와 운영 방식에 맞춰, 사용하지 않는 프로토타이핑 도구 기준을 제거하고 MCP 연결법, 도구/작업 선택 기준, AI 모션 제작 워크플로를 메인 가이드에 명확히 연결했습니다.

운영 기준 정리

  • 사용하지 않는 도구 기준 제거 — 실제 운영하지 않는 프로토타이핑 도구가 의사결정 기준에 섞이지 않도록 방향 확정/프로토타입 섹션에서 제외
  • 프로토타입 기준 재정리 — FigJam, Figma 프로토타입, HTML/CSS 인터랙션 시안, 코드 프로토타입/실제 개발 화면 기준으로 정리
  • Figma MCP 연결 설정 보강 — 홈 카드, 사이드바, 메인 가이드 본문에서 바로 접근 가능하도록 연결하고 첫 테스트 체크리스트 추가
  • 도구 / 작업 선택 가이드 정리 — 실제 설치 여부가 불명확한 스킬명 나열을 줄이고 탐색/정리/검증/전달 업무 기준으로 재작성
  • AI 모션 제작 워크플로 보강 — 레퍼런스 녹화, 넘버링, 수치화, 조정 가능한 미리보기, FE 전달 패키지까지 포함
0.6

방향 이탈 감지와 실전 운영 보강 2026.05.29

Figma 공식 리포트와 디자인팀 실험 기록에서 바로 운영 문서에 녹일 수 있는 항목을 선별해, 방향 확정 전 점검, QA 운영 구조, Figma MCP 실전 함정을 보강했습니다.

가이드 본문 보강

  • 방향 이탈(Drift) 감지 신호 — 피드백 반복, 설명하기 어려운 어긋남, 구현 drift, 방향 선택 지연의 4가지 신호와 대응 액션 추가
  • 방향 확정 전 체크 — 옵션 비교, 트레이드오프 명시, Figma/코드 현실화, 외부인 이해 가능 여부 확인 흐름 추가
