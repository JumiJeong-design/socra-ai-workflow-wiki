# Content Review

가이드 콘텐츠를 4가지 관점에서 검토한다.

## 검토 대상

- `ai-workflow-guide.html` (메인 가이드)
- `figma-mcp-traps.html` (Figma MCP 함정)

## 검토 관점

### 1. 사실 정확성
웹에 공개된 정보와 다른 내용을 찾는다:
- WCAG 기준 (SC 번호, Level, 수치)
- CSS 브레이크포인트와 기기 실제 뷰포트 너비
- Figma MCP 동작과 한계
- AI 도구 (Claude Code, Cursor) 실제 기능

### 2. 내용 완결성
디자이너가 실제 작업할 때 필요한데 빠진 내용을 찾는다:
- 단계별 지시가 너무 막연하지 않은가
- 예시가 바로 쓸 수 있는 수준인가
- 이 단계만 읽으면 다음에 뭘 해야 할지 아는가

### 3. 일관성
섹션 간 충돌을 찾는다:
- 같은 개념을 다르게 설명하는 곳
- 용어 혼용 (AI Workflow vs ai-workflow.md 등)
- 서로 모순되는 조언

### 4. 독자 경험
디자이너가 실무 중 문서를 열었을 때의 경험:
- 10초 스캔으로 이 섹션이 뭔지 파악 가능한가
- callout, highlight, card 컴포넌트가 의도에 맞게 쓰였는가
- orange callout(⚠️)이 실제 경고에만 쓰였는가

## 보고 형식

파일 · 섹션 기준으로 정리한다. 각 항목에:
- 어떤 문제인지
- 왜 독자에게 문제가 되는지
- 수정 제안 (한 줄)

잘 된 부분은 건너뛰고 실제 문제만 보고한다.
