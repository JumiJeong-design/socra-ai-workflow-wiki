0.6

방향 이탈 감지와 실전 운영 보강 2026.05.29

Figma 공식 리포트와 디자인팀 실험 기록에서 바로 운영 문서에 녹일 수 있는 항목을 선별해, 방향 확정 전 점검과 QA/MCP 실전 함정을 보강했습니다.

가이드 본문 보강

  • 방향 이탈(Drift) 감지 신호 — 피드백 반복, 설명하기 어려운 어긋남, 구현 drift, 방향 선택 지연의 4가지 신호와 대응 액션 추가
  • 방향 확정 전 체크 — 옵션 비교, 트레이드오프 명시, Make/코드 현실화, 외부인 이해 가능 여부 확인 흐름 추가
  • 충실도별 프로토타입 선택 기준 — FigJam, Figma, Make, 코드 프로토타입을 언제 쓸지 정리
  • 디자이너 + AI QA 운영 구조 — 디자이너 수정 후 개발자 검수 흐름과 QA R&R 기준 추가
  • AI 모션 제작 프롬프트 흐름 — 레퍼런스 녹화, 넘버링, 수치화, 조정 가능한 미리보기 요청 방식 추가

Figma MCP 함정 추가

  • T7 · 작업 페이지 지정 누락 — 여러 페이지 파일에서 setCurrentPageAsync로 컨텍스트를 먼저 맞추는 원칙 추가
  • T8 · 로컬 폰트 접근 단정 — 원격 플러그인/로컬 서버 방식에 따라 폰트 접근 가능성이 달라짐을 조건부로 정리
  • T9 · Variables 읽기 실패 단정 — 연결 방식과 권한, semantic token 구조를 먼저 확인하도록 정리
0.5

Figma MCP 함정 모음 2026.05.28

실제 운영 중 마주친 Figma MCP 작업 함정을 별도 페이지로 정리했습니다. 앞으로도 새 함정이 발견될 때마다 추가됩니다.

신규 페이지

  • Figma MCP 작업 함정 모음 — 실전 레퍼런스 섹션 신설, 총 6개 함정 수록

수록 함정

  • T1 · 패널 내부 raw 프레임 실수 — 기존 컴포넌트 있을 때 인스턴스 사용 필수
  • T2 · variant 일부만 수정 — 부모 프레임 metadata 먼저 조회, 전체 variant 적용
  • T3 · vectorPaths 한계 — H/V/a 커맨드 및 소수점 좌표 파싱 실패, 오픈소스 SVG 직접 주입 불가
  • T4 · createNodeFromSvgAsync 미지원 — MCP 환경에서 해당 메서드 없음
  • T5 · combineAsVariants 위치 이슈 — combine 후 자식 노드 위치 직접 설정 필요
  • T6 · AI 외부 메모리 vs 프로젝트 문서 — ~/.claude 같은 외부 메모리 대신 프로젝트 docs에 기록
0.4

스킬 선택 가이드 2026.05.28

디자이너가 설치된 Figma / AI 스킬을 실제 업무 흐름 안에서 선택할 수 있도록, 실행 플랜 영역에 스킬 선택 가이드를 추가했습니다.

신규 섹션

  • 스킬 선택 가이드 — 실제 업무 활용 다음에 배치해 탐색, 정리, 검증, 전달 단계에서 어떤 스킬을 쓸지 안내
  • 업무 흐름별 선택 순서 — 탐색/초안 → 시스템 적용 → 드리프트 점검 → 스펙화 → 전달/동기화 흐름으로 재구성
  • 상황별 추천 표 — 신규 화면, 기존 화면 정리, 컴포넌트 스펙, 정책 시각화, 토큰 동기화, 리뷰 자료 제작 상황별 첫 번째 스킬 제안

정리한 내용

  • 사이드바 버전 — v0.4로 업데이트하고 cache busting 값을 갱신
  • 가이드 상단 버전 배지 — v0.4로 업데이트하고 요약 changelog 항목 추가
  • 스타일 토큰 — 색상, 보더, 코드 토큰 값을 CSS 변수 기준으로 정리
0.3

제품 유형별 확장 2026.05.28

Core 가이드 위에 제품 유형에 따라 선택 적용할 수 있는 확장 섹션 5개 추가. 다크모드를 가이드 사이트 자체에 구현해 레퍼런스로 활용.

신규 섹션

  • 제품 유형 자가진단 — 4가지 질문으로 어떤 확장 섹션이 필요한지 안내
  • 다크모드 대응 — 시맨틱 토큰 레이어, CSS custom properties 구조, FOUC 방지, AI 규칙
  • 다국어(i18n) 설계 — 텍스트 하드코딩 금지, locale 파일 구조, 레이아웃 주의사항, AI 컨텍스트 문서
  • 반응형 + 기기 QA — 브레이크포인트 기준표, iOS/Android 엣지케이스, 터치 타겟, 텍스트 확대 대응
  • 인터랙션 / 모션 품질 — 모션 원칙 3가지, easing/duration 기준, prefers-reduced-motion 처리
  • 브랜드 시각 언어 — AI 이탈 패턴 5가지, 브랜드 토큰 정의, 결과물 체크리스트

구현

  • 다크모드 가이드 사이트 적용 — CSS 다크 테마 토큰, 토글 버튼 (사이드바 + 모바일 탑바), localStorage 저장, prefers-color-scheme 기본값
  • 사이드바 단일 파일 관리 — sidebar.html fetch 주입 구조로 모든 페이지 동기화
  • 모바일 topbar 높이 개선 — 52px → 60px, 햄버거 여백 확보

구조 개선 2026.05.28

초안(v0.1) 이후 첫 번째 보강 버전. 섹션 추가, 콘텐츠 정확도 개선, 멀티 AI 에디터 지원, 디자인·반응형 수정이 포함됩니다.

추가된 섹션

  • 빠른 시작 (⚡) — 5단계·예상 소요 시간 포함. 처음 접하는 사람이 바로 실행할 수 있도록 가이드 앞단에 배치
  • AI 에디터 연결 설정 — CLAUDE.md, .cursor/rules/, .windsurfrules, .github/copilot-instructions.md 4종 에디터 동시 지원 방법 + docs/ 폴더 구조 예시

콘텐츠 개선

  • Figma 포지셔닝 수정 — "Figma 줄이기" 방향에서 "팀 규모·프로세스에 따라 결정" 으로 뉘앙스 조정. 탐색 도구 vs 결과 산출물 모두 가능하다는 관점 반영
  • 용어 설명 인라인 추가 — SOT(Single Source of Truth), visual regression, WCAG 등 맥락 내 간단 주석
  • Step 5 프롬프트 단계적 빌딩 플로우 — 컨텍스트 → 역할 → 제약 → 출력형식 4단계 구조 추가
  • 핵심 원칙 실패 사례 예시 — 원칙 3에 "AI 결과를 그대로 넘겼을 때" 구체적 실패 케이스 3개 추가
  • 환경 사전 조건 콜아웃 — 개요 섹션에 도구 준비 상태 체크리스트 추가

디자인 · 반응형 수정

  • 모바일 안전 영역 처리 — iOS Dynamic Island, 홈 인디케이터, Android punch-hole 대응. env(safe-area-inset-*) + 100dvh + viewport-fit=cover
  • 콜아웃 박스 좌측 컬러 보더 제거 — border-left: 3px solid 디자인 폐기. green/orange 변형은 배경 틴트로만 구분
  • doc-table 폰트 정규화 — th 11px→12px, td 13.5px→15px (본문 텍스트와 통일)
  • footer-quote 좌측 보더 제거 — 콜아웃과 동일 기준 적용

최초 작성 2026.05.27

프로덕트 디자이너를 위한 AI Workflow 운영 문서 구축 가이드 초기 버전. 홈 페이지(인덱스) + 전체 가이드 문서로 구성.

추가된 내용

  • 홈 페이지 (index.html) — 가이드 사용법, 지금 할 수 있는 액션, 단계별 빠른 참조
  • 개요 — 01 문서 목적, 02 핵심 관점
  • 배경 — 현장 이슈와 AI 역할 구분, 문서 레이어 구조
  • AI 전략 — 제품 단계별 AI 활용 전략 (0→1 / 1→10 / 10→100)
  • 단계별 가이드 — Step 0~8 (탐색, Product Context, Design System, AI Workflow, Agent Rules, Prompts, Review/QA, Handoff, Release)
  • 실행 플랜 01~05 — 구축 우선순위, MVP 문서 4개, 실제 업무 활용, 핵심 원칙, 2주 실험 계획
  • 참조 — AI 도입 시 디자이너 일정 산정 가이드
  • 디자인팀 AI 활용 실험 기록 DB 링크 및 3종 템플릿 안내

설계 기준

  • Socra AI 디자인 시스템 라이트 테마 적용 (SocraPurple #664FFF 기반)
  • 사이드바 고정 + scroll-spy 현재 위치 하이라이트
  • 반응형: 모바일(≤640px) 햄버거 + 슬라이드 드로어
  • 폰트: Pretendard Variable, 본문 16px / 라인높이 1.75
  • CSS/JS 외부 파일 분리, 인라인 스타일 없음
