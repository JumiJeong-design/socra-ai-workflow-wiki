레거시 플로우
장점
  • ✓ Figma만 알면 됨 → 진입장벽 낮고 초기 세팅 비용 없음
  • ✓ 도구 간 동기화 신경 쓸 필요 없음 → 단순함
  • ✓ 팀 전체가 Figma만 보면 되므로 커뮤니케이션 도구가 단일화됨
단점
  • ✕ 인터랙션·데이터 상태는 실제 화면 들어가야 확인 → 후반부에 문제 발견
  • ✕ 컴포넌트 문제와 UX 문제를 화면 QA에서 한꺼번에 → 원인 구분 어려움
  • ✕ QA 발견이 기록 안 되고 수정으로 끝남 → 다음 이터레이션에 같은 문제 반복
  • ✕ 기준이 사람 머릿속에만 존재 → AI·개발자에게 매번 다시 설명
신규 플로우 (AI 기반)
장점
  • ✓ 이터레이션이 사이클마다 빨라짐 — 기준이 쌓일수록 다음 작업은 그 위에서 시작. AI가 발전할수록 활용도 높아짐
  • ✓ Storybook에 검수 기준 저장 → 컴포넌트 문제 미리 잡고 화면 QA는 UX 흐름에 집중
  • ✓ QA 발견 → story 승격 → 재사용. 같은 문제 반복 줄어듦
  • ✓ Worklog가 AI context source → 지금 쌓는 기록이 미래 AI 활용의 기반
단점
  • ✕ 초기 세팅 비용 큼 → Figma·Git·Storybook 각각 기준 정리 필요
  • ✕ 도구 간 sync가 어긋나면 오히려 혼란 → 지속적인 관리 부담
  • ✕ 화면 설계 단계 적용 여부는 아직 미검증 → 컴포넌트 이상의 범위는 테스트 필요
